## Phase Goal

**As a** game developer using DSM, **I want to** keep the save system's shared state and save/load pipeline safe under concurrent access and invalid input, **so that** my players never lose or corrupt their save data.

## Acceptance Criteria (Slice 3)
- [ ] Slot names outside `[A-Za-z0-9_-]`, containing `..`, path separators, or Windows-reserved device names are rejected with `ArgumentException`.
- [ ] Malformed JSON on load logs a warning (slot name + error) and falls back to defaults without throwing.
- [ ] A component-less widget prefab logs an error and is skipped.
- [ ] The full DSM Foundation EditMode suite is green in the Unity Test Runner (human-verified).
